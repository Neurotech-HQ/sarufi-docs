# Workspaces

A **workspace** is the top-level container for everything on the platform. All chatbots, flows, knowledge bases, team members, and settings belong to a workspace. Think of it as your project folder or your team's shared environment.

---

## What Workspaces Are For

Use separate workspaces to:
- **Isolate clients or projects** - one workspace per client keeps things clean
- **Separate environments** - e.g., a "Production" workspace and a "Staging" workspace
- **Collaborate with different teams** - each workspace has its own member list and roles

---

## Creating a Workspace

1. Go to the **Workspaces** page (the platform home page after login).
2. Click **Create Workspace**.
3. Enter:
   - **Name** - required, up to a few words (e.g., "Customer Support")
   - **Description** - optional, helps teammates understand the workspace's purpose
4. Click **Create**.

You land on the new workspace's **Overview** page.

:::note
Your plan controls how many workspaces you can create. If the **Create Workspace** button shows an upgrade prompt, you have reached your plan's workspace limit. See [Billing & Plans](./billing-and-plans).
:::

---

## The Workspace Overview Page

The Overview page gives you a snapshot of activity in the workspace:

- **Stats cards** - total chatbots, conversation flows, meta flows, and team members
- **Chatbot list** - all chatbots in this workspace with their active/inactive status
- **Quick actions** - create a new chatbot from here

---

## Switching Between Workspaces

1. Click the **back arrow** or the platform logo in the sidebar to return to the **Workspaces** page.
2. Click on any workspace card to open it.

Your session remembers which workspace you were last in, so you can quickly pick up where you left off.

---

## Workspace Settings Overview

Access workspace settings from the sidebar: click **Settings**.

The settings page has four tabs:

| Tab | What you can do |
|-----|----------------|
| **General** | Update workspace name and description |
| **Members** | Invite teammates, assign roles, remove members |
| **API Keys** | Create and revoke API keys for programmatic access |
| **Danger Zone** | Permanently delete the workspace (owner only) |

For full details on each tab, see [Team & Settings](./team-and-settings).

---

## Deleting a Workspace

:::warning
Deleting a workspace is permanent and irreversible. All chatbots, flows, knowledge bases, and data within it will be deleted.
:::

Only the **workspace owner** can delete a workspace.

1. Go to **Settings** → **Danger Zone** tab.
2. Read the warning carefully.
3. Type the workspace name exactly to confirm you intend to delete it.
4. Click **Delete Workspace**.

The workspace and all its contents are permanently removed.
